Grafana2PDF Privacy Policy

Effective date and last updated: 9 July 2026

Grafana2PDF processes user-selected Grafana dashboard content locally in the browser solely to generate PDF and PNG files. The extension does not send dashboard content, authentication information, analytics, or usage data to the developer or to third parties.

Purpose and scope

Grafana2PDF has a single purpose: exporting dashboards selected by the user from a Grafana server as local full-page PDF and PNG files. This policy applies to the Grafana2PDF browser extension.

Information processed

The extension processes only the information needed to perform an export:

  • The user-selected Grafana server origin and base path.
  • Grafana folder and dashboard identifiers, titles, URLs, and visible dashboard content.
  • User-supplied run identifiers, time ranges, output settings, and dashboard variables.
  • Captured dashboard pixels used to construct the requested PDF and PNG files.
  • Local execution status, errors, timing, dimensions, and output filenames used for progress logs and interrupted-run recovery.

Authentication

The extension uses the browser's existing signed-in Grafana session. It does not request or store a Grafana password, API token, or other login credential. The browser may send the user's existing session cookies directly to the selected Grafana server, as it normally does when the user visits Grafana. Grafana2PDF does not read, copy, or record those cookies, authentication headers, or response bodies.

How information is used

The processed information is used only to:

  • Find dashboards that the signed-in user can access.
  • Open and render the dashboards selected by the user.
  • Generate PDF, PNG, JSON log, and CSV summary files in the browser's Downloads directory.
  • Show export progress and recover execution logs after an interrupted export.

Network communication and sharing

Grafana2PDF communicates only with the Grafana origin selected by the user. It has no developer server, analytics, telemetry, advertising, tracking, or remote-code service. Information processed by the extension is not sold, transferred, or shared with the developer or any third party. Output files are saved locally through the browser's download system.

Local storage and retention

Pending job settings and execution journals are stored locally in the extension's browser storage. Dashboard variable values can be present in a pending job while it is being executed, but persisted run logs record variable names only. After final log files are downloaded successfully, the extension removes that execution's pending job and journal. Interrupted runs or failed log downloads can remain until the user downloads the recovered logs or dismisses the recovery notice.

Removing the extension clears its extension storage according to the browser's normal behavior. Files already saved in the Downloads directory remain until the user deletes them.

User control

Exporting starts only after a user action. Users choose the Grafana server, dashboards, time ranges, variables, and output formats. Users can cancel an export, dismiss locally retained recovery data, revoke the optional permission for a Grafana origin in browser settings, delete downloaded files, or uninstall the extension.

Limited use

Grafana2PDF's use of information is limited to providing and improving its disclosed dashboard-export functionality. The extension does not use information for advertising, credit, lending, or any unrelated purpose.

Changes to this policy

This policy may be updated when the extension's functionality or data practices change. Material changes will be reflected on this page by updating the date above.

Contact

Privacy questions can be submitted to the maintainer through the rithomp GitHub profile.

Privacy-policy website hosting

This policy page is hosted by GitHub Pages. GitHub may process standard website request information under the GitHub General Privacy Statement. Visiting this page is separate from using Grafana2PDF; the extension does not contact GitHub.
